📊 Current Connector Usage

Understanding which connectors are actively used and how teams currently work with integrations

1

Which existing connectors are you currently using?

Identify active connectors in Workflows and Apps

Required
When
After user completes Integration Hub tour or views Unified/Workflow/Apps tabs for first time (30 seconds after page load)
Why
Establish baseline of current connector adoption and understand existing integration patterns before asking about gaps
Options
  • Salesforce
  • Google Sheets
  • Microsoft Excel
  • SAP
  • Other (open text)
User Flow Type
Flow + Multi-Select
Interactive flow with checkbox selection and autocomplete search for connector names
Location
Main Connections Page
After user views the main Connections list (Unified tab)
Timing
30 seconds after page load if user has viewed ≥5 connections; OR after tab switching
2

How easy or difficult was it to set up and use these connectors?

Evaluate ease of use and identify friction points

Required
When
Immediately after Q1 (within same flow, no delay)
Why
Understand setup pain points and identify UX improvements needed for connector creation workflow
Options
  • Very Easy (1)
  • Easy (2)
  • Neutral (3)
  • Difficult (4)
  • Very Difficult (5)
User Flow Type
NPS-Style 5-Point Scale
Single-question survey with emoji scale (😄 to 😢) for quick sentiment capture

Continue same flow without modal interruption
Branching

🔄 Connection Type Comprehension

Assessing user understanding of Unified, Workflow, and Apps connection types

3

Do you understand the difference between Unified, Workflow, and Apps connections?

Assess Unified connection architecture understanding

Required
When
After user clicks on Workflow or Apps tab (switching between tabs triggers this)
Why
Critical to understand if users comprehend the value of Unified connections (eliminates duplication) vs. creating separate Workflow/Apps connections
Options
  • Yes, I understand the difference
  • Somewhat, but confused about when to use each
  • No, I'm confused about the three types
  • Skip - already familiar
User Flow Type
Inline Flow + Branching
Single-select radio buttons with optional educational tooltip on hover showing "Unified = both Workflows & Apps"
Location
Tab Navigation Area
Appears as tooltip/callout near the 3 tabs (Unified, Workflow, Apps) when user hovers or clicks tab
Branching
If "No/Somewhat": Show Q4 (confusion follow-up); If "Yes/Skip": Jump to Q5
4

Have you been confused about which connection type to create?

Identify specific confusion points in connection type selection

Optional
When
Only shows if Q3 answer was "Somewhat confused" or "No" (conditional branching)
Why
Deep dive into specific UX/messaging gaps that cause confusion when creating new connections
Options
  • When choosing between Unified vs Workflow
  • When choosing between Unified vs Apps
  • Not sure if a connector supports Unified
  • Didn't understand the tooltip/guidance
  • Other (open text)
User Flow Type
Flow + Multi-Select
Multiple choice checkboxes (user can select multiple confusion areas)
Location
Inline (continuation of Q3 flow)
Same flow context, appears right after Q3 answer
Timing
Display immediately after Q3 (no delay)

🔍 Connector Discovery & Search

Understanding whether users can easily find the connectors they need

5

Is it easy to find the connector you're looking for?

Measure connector discoverability across tabs

Required
When
When user is in the "Select Connector" dropdown menu (during connection creation)
Why
Understand if users can discover needed connectors or if search/organization is confusing
Options
  • Very Easy - Found it immediately
  • Easy - Found it with one search
  • Neutral - Had to look around
  • Difficult - Took multiple searches
  • Very Difficult - Couldn't find it
User Flow Type
Toast/Modal NPS Scale
Small toast notification with 5-point scale appearing over the connector dropdown (non-intrusive)
Location
Inside Create Connection Modal

Timing
Appears 3 seconds after dropdown is opened (give user time to search first)

  • Search history

  • Show recently used first
  • Other (open text)

🎯 Missing Connectors & Actions

Identifying missing connectors and missing actions for existing connectors

7

What connectors and actions are you missing?

List missing connectors and missing actions for existing connectors

Required
When
After user completes connection setup or when viewing the Connections list (after ≥2 connections viewed)
Why
Identify roadmap priorities - which connectors are most requested and what actions are missing from existing connectors
Options
  • Missing connectors (open text with autocomplete from full connector list)
  • Missing actions for [selected connector] (e.g., "Update row" for Google Sheets)
  • Both (multi-step)
User Flow Type
Flow + Multi-Step
Step 1: Text input with autocomplete for missing connectors; Step 2: For each connector selected, ask what actions are missing
Location
Main Connections Page
Flow triggers on main Connections page (Unified tab) after viewing list
Timing
After user views ≥2 connections or after 1 minute on the page (whichever comes first)

⏰ Timeline & Business Impact

Understanding urgency and quantifying business value

11

What's your timeline to need these connectors/improvements?

Segment users by urgency

Required
When
Exit flow - when user is about to close/leave the Integrations page or after 5 minutes on page
Why
Identify critical/blocking needs vs. future enhancements - roadmap prioritization
Options
  • ASAP (blocking current projects)
  • High Priority (within 1-3 months)
  • Medium Priority (within 3-6 months)
  • Low Priority (exploring/future)
User Flow Type
Exit-Intent Modal
Modal that appears when user attempts to close/navigate away from page
Location
Full-Page Modal
Centered modal blocking page exit (user can dismiss with X or answer question)
Timing
Trigger on exit-intent (mouse to close/back button) OR after 5 minutes on page, whichever comes first
12

What's the business impact if you can't get these connectors/improvements?

Required
When
Immediately after Q11 (continue same exit-intent modal)
Why

Options
  • Time: Manual workarounds consume X hours/week
  • Cost: Would save ~$X in annual costs
  • Revenue: Would enable ~$X in new revenue
  • Risk: Would reduce errors/delays that impact customers
  • Other (open text)

Multi-Select Checkboxes + Open Text
Multiple choice + optional number input for quantified impact
Location
Same Exit-Intent Modal
Appears below Q11 in the same modal
Branching
