Nintex · Connector Strategy
in progress

Connector Acceleration

Using User Flow to get feedback from users on missing connectors, missing actions, finding connections, creation ease, and awareness

🎯 Research Priorities

PRIMARY (70%)

  • Missing connectors (which ones, why)
  • Missing actions in existing connectors
  • Business criticality/importance
  • Impact & timeline

SECONDARY (30%)

  • Ease of creating connections
  • Ease of finding connectors
  • Awareness of unified behavior

👤 Journey 1: First-Time User (Creates First Connection) SECONDARY (30%)

User learns about connection types first, then creates their first connection

User lands on empty Connections page ↓ L1: Launcher flow appears: "Create your first connection now! Unified (works in Workflows & Apps, all environments) | Workflows (Workflows only) | Apps (Apps only)" ↓ User clicks "Create Connection" button → Selects connection type from dropdown ↓ User selects connector → Fills credentials ↓ Q1: "How easy was it to create this connection?" (Post-success toast) ↓ Q1a: "What was difficult?" (Conditional, if Q1 rating ≤6) ↓ Q2: "How easy was it to find the connector you needed?" (Same toast) ↓ Q2a: "What would help find connectors more easily?" (Conditional, if Q2 rating ≤6)
L1

Create your first connection now!

Educate about connection types before creation

Optional
When
User lands on empty Connections page
Why
Educate users about the three connection types (Unified/Workflows/Apps) before they attempt to create, increasing awareness and preventing confusion

1

How easy was it to create this connection?

Understand creation UX friction

Required
When
Immediately after the success toast message appears (connection created successfully)
Why
Capture setup UX friction while experience is fresh - inform UI/UX improvements
Type
Likert Scale (1-10) + Text Area
Numeric 1-10 rating scale with optional text area for user to add details if needed
Options
  • 1 - Very Difficult
  • 2-3 - Difficult
  • 4-6 - Neutral
  • 7-8 - Easy
  • 9-10 - Very Easy
Branching
  • If ≥7 (Easy): Skip Q1a → Proceed directly to Q2 (finding connectors)
  • If 4-6 (Neutral/Some friction): Show Q1a "What was difficult?" → Then proceed to Q2
  • If 1-3 (Very Difficult): Show Q1a "What was difficult?" → Route to Q8 (skip Q2-Q9, escalate to customer dev call)
1a

What was difficult about creating this connection?

Understand specific friction points

Conditional
When
Triggered only if Q1 rating ≤6 (4-6: moderate friction, 1-3: critical friction)
Why
Diagnose specific UX problems preventing smooth connection creation - inform priority fixes
Type
Follow-up Flow + Open Text
Open text input for user to describe what was difficult
Options
Open text input: "What was the most difficult part of creating this connection? (e.g., entering credentials, understanding the form, etc.)"
Branching
If Q1 was 4-6: Proceed to Q2 after answer; If Q1 was 1-3: Proceed to Q8 (customer dev call) after answer
2

How easy was it to find the connector you needed?

Understand connector discovery UX

Required
When
After successful creation (appears after Q1 flow completes)
Why
Identify search/discovery friction - critical for improving connector findability
Type
Likert Scale (1-10) + Text Area
Numeric 1-10 rating scale with optional text area for user to add details if needed
Options
  • 1 - Very Difficult (couldn't find it)
  • 2-3 - Difficult (multiple searches)
  • 4-6 - Neutral (had to browse)
  • 7-8 - Easy (one or two searches)
  • 9-10 - Very Easy (found immediately)
Branching
If ≥7: Skip Q2a → Proceed to Q4; If ≤6: Show Q2a "What would help find connectors?"
2a

What would help you find connectors more easily?

Understand discovery improvement opportunities

Conditional
When
Triggered only if Q2 rating ≤6 (had difficulty finding connector)
Why
Identify specific improvements needed for connector discovery UX - inform search/filter/navigation enhancements
Type
Follow-up Flow + Open Text
Open text input for user suggestions on improving discovery
Options
Open text input: "What features or improvements would help? (e.g., better search, filters, categories, suggestions)"
Branching
After answer → Journey 1 complete (first-time user flow ends)

👥 Journey 2: Existing User - PRIMARY FOCUS: MISSING GAPS Primary 70%

User has 3+ connections - we focus on gaps: missing connectors, missing actions, business impact

User views Connections list (has ≥3 active connections) ↓ Q4: "Which connectors are you CURRENTLY using regularly?" (Multi-select) ↓ Q4a: "For the connectors you're using, what actions/features are missing?" (Open text per-connector) ↓ Q5: "What connectors are you MISSING that you need?" (Open text + autocomplete from catalog) ↓ Q6: "How critical are these missing connectors, and what's the business impact?" (Rating + Impact) ↓ Q7: "Timeline - when do you need these?" (Urgency)
4

For the connectors you're using, what specific actions or features are missing?

PRIMARY: Identify missing actions in currently used connectors

Required
When
Launcher Flow - Step 2
After Q4 answer within Launcher — continuation in the same launcher flow
Why
Users already rely on these connectors — missing actions may block workflows. Critical to understand what features and actions would unlock more value in existing connectors (easier wins than new connectors)
Type
Flow + Open Text (Per-Connector)
Progressive disclosure - for each connector selected in Q4, ask what actions are missing
Options
Open text input for each selected connector: "What specific actions or features are missing for [Connector Name]? (e.g., Update action, Real-time sync, Bulk operations, etc.)"
Branching
Always shows after Q4; proceeds to Q5 after completion
5

What connectors are you MISSING that you need?

PRIMARY: Identify missing connectors for roadmap

Required
When
Launcher Flow - Step 3
After Q4a answer within Launcher — continuation in the same launcher flow
Why
THE MOST CRITICAL DATA - which connectors drive adoption decisions and roadmap value
Type
Flow + Open Text with Autocomplete & Free-Form Entry
Text input with searchable connector list (197 connectors) dropdown + ability to add custom connector names not in the list
Options
Users can select from new Nintex connector list (197 connectors) via autocomplete search, OR manually enter connector names not yet in the product roadmap - multiple entries allowed
Branching
If ≥1 connector selected → Show Q6 (criticality & business impact); If 0 → Skip to Q7 (timeline)
6

How critical are these missing connectors, and what's the business impact?

PRIMARY: Understand urgency and business value for roadmap prioritization

Required
When
Launcher Flow - Step 4
After Q5 answer within Launcher (if ≥1 connector selected) — continuation in the same launcher flow
Why
Capture both criticality LEVEL and real BUSINESS IMPACT - prioritize roadmap based on what matters most: time saved, revenue, errors prevented, compliance, etc.
Type
Flow + Multi-Part (Rating + Open Text)
Part 1: Single select rating per connector; Part 2: Open text for business impact details
Options
Part 1 - Criticality Rating (Radio buttons per connector):
  • BLOCKING - Can't achieve our goals without it
  • CRITICAL - High priority, plan to use soon
  • IMPORTANT - Would improve efficiency
  • NICE TO HAVE - Would be useful someday
Part 2 - Business Impact (Open text): "What's the business impact if you don't have these connectors? (e.g., X hours/week of manual work, $X cost savings, Y% error reduction, compliance requirements, revenue impact, etc.)"
Branching
Always shows after Q5 (if ≥1 connector selected); proceeds to Q7 (timeline) after completion
7

When do you need these missing connectors?

Understand timeline and urgency

Required
When
Same Flow - Final Question
After Q6 answer (or directly after Q5 if user selected 0 connectors in Q5) — conclusion to the gaps flow
Why
Prioritize roadmap - understand which gaps are urgent vs. eventual
Type
Flow + Single Select
Radio buttons for timeline
Options
  • ASAP - Blocking immediate projects (within 2 weeks)
  • Soon - High priority (within 1-3 months)
  • This quarter - Planned (3-6 months)
  • Future - Exploring/not urgent
Timing
After gaps inventory complete

Last chance to capture data before user navigates away

User attempts to leave page (close, back, navigate away) ↓ Q8: "Can we schedule a 15-minute call to learn more about your integration needs?" (Convert to customer development call)
8

Convert research to deeper engagement

Optional
When

Options

  • Maybe later
  • No thanks
