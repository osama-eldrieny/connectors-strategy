Connector Acceleration

UX Research Specification: Integration Hub Discovery Interview based on realistic user journeys

👤 Journey 1: First-Time User (Empty Integrations Page)

User lands on Integration Hub for the first time with zero connections

User lands on empty page ↓ Q0: "Interested in unified connections?" (Launcher entry flow) ↓ User clicks "Create Connection" ↓ Q3: "Understand 3 connection types?" (Context when selecting) ↓ User completes setup ↓ Q1: "How easy was setup?" (Post-success toast/modal) ↓ (User may skip Q2 - has only 1 connection)
0

Are you interested in creating unified connections for your team?

Entry point for new users on empty Integrations page

Optional
When
User lands on Integration Hub with ZERO existing connections (empty state page)
Why
Gauge interest before user commits to setup journey; understand if unified connections concept resonates
Options
  • Yes, I want to create unified connections
  • Maybe later
  • No, I'll create Workflow/Apps connections separately
User Flow Type
Launcher (Optional Overlay)
Non-intrusive launcher that appears over empty state with an info icon next to "Create Connection"
Location
Empty State Message
Appears as launcher next to the "Create Connection" button on empty page
Timing
Page load, only if connection count = 0 AND not previously shown (once per session)
3

Do you understand what Unified connections do?

Check comprehension when user is actively creating

Required
When
User clicks on "Create Connection" dropdown and selects "Unified" vs "Workflow" vs "Apps"
Why
In-context question when user is making the actual choice - understand if messaging/tooltips are clear
Options
  • Yes - Unified works for both Workflows & Apps
  • Somewhat - I'm not 100% sure
  • No - I'm confused about the three types
  • Skip - Already know this
User Flow Type
Inline Flow + Radio Buttons
Single-select appearing inside "Create Connection" modal when type is selected
Location
Inside Create Modal
Appears between type selection and next step (choosing connector)
Branching
If "No/Somewhat": Show educational tooltip explaining Unified; If "Yes/Skip": Continue to connector selection
1

How easy was it to set up this connection?

Capture setup experience immediately after success

Required
When
Immediately after "Connection created successfully!" toast appears (post-success state)
Why
Capture UX friction while fresh - understand setup pain points before user moves on
Options
  • 😄 Very Easy
  • 🙂 Easy
  • 😐 Neutral
  • ☹️ Difficult
  • 😢 Very Difficult
User Flow Type
NPS Toast + Emoji Scale
Non-intrusive toast overlay with emoji scale showing success message
Location
Success Toast (Bottom Right)
Appears above the green "Connection created successfully" toast
Branching
If ≥4 (Difficult): Show follow-up "What was difficult?"; If 1-2 (Easy): Skip to connections list

👥 Journey 2: Existing User (1+ Connections Already Created)

User returns to Integration Hub with active connections and is exploring

User views Connections list (≥1 connections) ↓ Q2: "Which connectors are you using?" (Flow after 2+ sec viewing list) ↓ User clicks tabs (Unified/Workflow/Apps) ↓ Q3: "Understand 3 types?" (Context when switching tabs) ↓ User tries to create another connection ↓ Q5: "Can you find connectors easily?" (During connector dropdown search) ↓ User explores page ↓ Q6: "How often use connectors?" (After 2+ min on page) Q7: "What connectors missing?" (After viewing multiple connections OR unsuccessful search)
2

Which of these connectors are you actually using regularly?

Understand active connector adoption

Required
When
User views Connections list with ≥2 connections; after viewing list for 2+ seconds (auto-trigger)
Why
Understand which connectors are actually used vs. just created; identify adoption patterns
Options
Checkboxes for each visible connector in their list (e.g., Salesforce, Google Sheets, etc.) + "Other" option
User Flow Type
Flow + Multi-Select Checkboxes
Inline flow appearing below connections list with checkboxes for each connector
Location
Below Connections Table
Appears after connections list view, no modal - inline with page content
Timing
Auto-trigger after viewing list for 2+ seconds (only if ≥2 connections exist, once per session)
3

Do you understand the difference between Unified, Workflow, and Apps connections?

Check comprehension as they manage multiple types

Optional
When
User clicks on Workflow or Apps tab (tab switching action) - context-aware trigger
Why
They're actively viewing different connection types - perfect moment to understand if they know WHY they exist
Options
  • Yes - I understand the difference
  • Somewhat - Confused about when to use each
  • No - I'm confused about all three types
  • Skip - Already know this
User Flow Type
Tooltip + Radio Buttons
Tooltip that appears near tab navigation with single-select options
Location
Near Tab Navigation (Unified/Workflow/Apps)
Appears above tabs or as tooltip when user hovers/clicks tabs
Branching
If "Somewhat/No": Show educational callout; If "Yes/Skip": Dismiss immediately
5

Can you easily find the connector you're looking for?

Test discoverability during active search

Required
When
User opens connector dropdown in "Create Connection" modal and starts typing/searching (in-context)
Why
Capture UX friction at the moment it happens - while they're actively searching for a connector
Options
  • Very Easy - Found immediately
  • Easy - One search found it
  • Neutral - Had to browse
  • Difficult - Multiple searches
  • Very Difficult - Couldn't find it
User Flow Type
Toast NPS Scale
Small toast appearing above the dropdown with emoji scale
Location
Inside Create Modal - Over Dropdown
Toast appears when dropdown is open for 3+ seconds
Branching
If ≥3: Show Q6 (improvements needed); If 1-2: Continue to next step
6

What improvements would help you find connectors faster?

Identify search/filter feature requests

Optional
When
Only shows if Q5 answer was "Neutral" or worse (≥3 on difficulty scale) - conditional trigger
Why
Get specific feature requests from users who struggled - actionable UX improvements
Options
  • Filter by "Unified-supported" only
  • Filter by use case/industry
  • Search history / Recent connectors
  • Group by category
  • Other (open text)
User Flow Type
Inline Flow + Multi-Select
Multi-checkbox options appearing in same modal below the dropdown
Location
Same Create Modal - Below Dropdown
Appears below the connector dropdown search field
Timing
Appears only if Q5 difficulty ≥3, within same session
7

What connectors or actions are you missing?

Identify roadmap gaps and priorities

Required
When
After user has viewed/used connections for 5+ minutes OR attempted to find connector and failed
Why
After hands-on experience, users can articulate what's missing from their workflow
Options
Open text input with autocomplete from full connector catalog + second field for "Missing actions for X connector"
User Flow Type
Flow + Multi-Step Form
Two-step flow: (1) Missing connector search, (2) For each selected - missing actions
Location
Main Connections Page
Appears inline below connections list OR as small modal overlay
Timing
Auto-trigger after 5+ minutes on page viewing connections, or manual trigger if search fails

⚙️ Journey 3: Advanced User (Multi-Environment/Multi-Tenant)

User managing connections across multiple environments (Dev, Test, Prod) or sharing with teams

User switches tenant/environment (Dev → Test → Prod) ↓ Q8: "How do you move connections between environments?" (Context on tenant dropdown) ↓ Q9: "Is export/import process clear?" (Conditional if they use it) ↓ User clicks connection options (menu/sharing) ↓ Q10: "Share connections with team?" (Connection context menu)
8

👋 Journey 4: Exit-Intent (User Leaving)

User attempts to close/navigate away from Integrations page

User moves mouse to close tab / clicks back / navigates away ↓ Q11: "Timeline for improvements?" (Exit-intent modal blocks action) ↓ Q12: "Business impact?" (Same modal - continuation) ↓ Q13: "Willing to be reference customer?" (Same modal - final question)
11

What's your timeline to need these improvements?

Prioritize roadmap based on urgency

Required
When
Exit-intent trigger: User moves to close browser tab, presses back button, or navigates away from page
Why
Last chance to capture high-value research data - segment users by urgency level
Options
  • ASAP - Blocking my projects
  • High - Within 1-3 months
  • Medium - Within 3-6 months
  • Low - Exploring/future
User Flow Type
Exit-Intent Modal (Full Page)
Centered modal blocking page exit with a close button (X) to dismiss
Location
Full-Page Modal Overlay
Centered modal that intercepts exit action
Timing
Trigger on: mouse moves to close tab, back button, or navigation away (not on page refresh)
12

What's the business impact if you can't get these improvements?

Quantify value and ROI

Required
When
Immediately after Q11 answer - same exit-intent modal, continues the flow
Why
Quantify impact for roadmap justification - most valuable research data at exit point
Options
Multi-select: "Time spent on manual work", "Cost savings opportunity", "Revenue impact", "Risk/error reduction", "Team adoption blocker"
User Flow Type
Exit-Intent Modal - Multi-Select
Checkboxes for impact categories + optional number input for specific amounts
Location
Same Modal - Below Q11
Appears below Q11 options in same modal
Branching
If "Time" selected: Show "Hours/week" input; If "Cost": Show "$" input; If "Revenue": Show "$" input
13

Optional
When

Why
Identify product advocates and case study candidates before user leaves
Options
  • Yes, absolutely
  • Maybe - ask later
  • No thanks
User Flow Type
Exit-Intent Modal - Final CTA
Radio buttons + conditional email capture if "Yes" selected
Location
Same Modal - Final Section

Branching
